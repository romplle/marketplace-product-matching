# Marketplace Product Matching

Решение задачи бинарной классификации матчинга предложений продавцов с товарами ассортимента. Реализована финальная часть пайплайна матчинга: для каждой пары (offer — good) модель принимает решение, является ли пара матчем (1) или нет (0). Использованы эмбеддинги изображений и названий+атрибутов, PCA-снижение размерности, десятки метрик сходства, продвинутая feature engineering и pseudo-labeling.

## Описание задачи

В соревновании необходимо решить финальный этап пайплайна матчинга:

- на вход подаётся пара объектов: **товар продавца (offer)** и **товар маркетплейса (goods)**;
- для каждой пары уже заранее найдены кандидаты на матч;
- требуется классифицировать, является ли пара совпадением или нет;
- качество оценивается по **F-score**.

Данные включают:
- табличные признаки: цена, категория, длина текстов, score от рескоринговой модели;
- эмбеддинги изображений;
- эмбеддинги текста (название + атрибуты).

## Подход к решению

Решение построено как классический табличный ML pipeline с использованием мультимодальных признаков.

Основная идея:
1. загрузить train/test;
2. подцепить эмбеддинги для offer и goods;
3. уменьшить размерность эмбеддингов через PCA;
4. построить признаки сходства между offer и goods;
5. обучить `CatBoostClassifier`;
6. подобрать порог классификации по `F1`;
7. улучшить модель через псевдоразметку уверенных объектов из test.

## Подготовка признаков

### 1. Работа с эмбеддингами
Для каждого объекта использовались:
- текстовые эмбеддинги;
- image-эмбеддинги.

Для снижения размерности:
- для текстовых векторов использовался `PCA` до **32** компонент;
- для image-векторов использовался `PCA` до **64** компонент.

Это позволило:
- уменьшить размерность признаков;
- сделать обучение модели более стабильным.

### 2. Метрики расстояния и сходства
Для пары `offer` и `goods` были рассчитаны признаки сходства:

- `manhattan`
- `euclidean`
- `cosine`
- `correlation`
- `canberra`
- `pearson`
- `spearman`
- `chebyshev`

Эти метрики считались отдельно:
- для текста;
- для изображений.

Также были добавлены попарные произведения:
- `title × image`
- `attrs+title_score × distance`

### 3. Числовые и инженерные признаки
Дополнительно использовались:

- `price_diff` — абсолютная разница цен;
- `price_ratio` — отношение цен;
- `log_offer_price`, `log_goods_price`;
- `log_price_diff`;
- `price_diff_log_ratio`;
- `cat_mean_target` — среднее значение target по категории;
- бинарные флаги:
  - наличие обоих изображений;
  - сильно ли offer дешевле goods;
  - сильно ли offer дороже goods;
- `price_ratio_rank_in_offer` — ранг пары среди кандидатов одного offer.

## Модель

В качестве классификатора использовался **CatBoostClassifier**.

Параметры обучения:
- `iterations = 1000`
- `eval_metric = 'F1'`
- `early_stopping_rounds = 100`
- `random_state = 42`

## Валидация и порог классификации

Для оценки качества:
- train был разбит на train/validation через `train_test_split(..., stratify=y)`;
- метрика — **F1-score**.

После обучения:
- получались вероятности класса `1`;
- подбирался оптимальный threshold по `precision_recall_curve`;
- финальное решение для test строилось по выбранному порогу.

Это важный момент: в задаче бинарной классификации с несбалансированными классами оптимальный порог часто даёт заметный прирост F1 по сравнению с фиксированным `0.5`.

## Pseudo-labeling

Для дополнительного улучшения качества применялось pseudo-labeling:

1. модель обучалась на train;
2. на test получались вероятности;
3. выбирались очень уверенные примеры:
   - `proba > 0.99` → positive
   - `proba < 0.01` → negative
4. эти примеры добавлялись в train с пониженным весом;
5. модель переобучалась на расширенной выборке.

Такой подход помогает использовать информацию из неразмеченного test и может улучшить обобщающую способность модели.

## Результаты

Финальная модель:
- использует мультимодальные эмбеддинги;
- сочетает PCA-признаки, distance-features и price-features;
- обучается на CatBoost;
- использует подбор порога и псевдоразметку.

Итоговый Score на Kaggle: **0.92651648** (5 место).
